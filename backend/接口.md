### 搜索职位
GET /api/jobs/search
参数：
- keyword：关键词
- location：地点
- education：学历要求
- positionType：职位类型
- salary：薪资范围

### 发布新职位

请求头：
```

```
请求体：
```json
{
    "title": "Java开发工程师",
    "location": "北京",
    "salary": "15k-25k",
    "education": "本科",
    "experience": "3-5年",

    "description": "职位描述...",
    "requirements": "任职要求...",
    "benefits": "职位福利..."
```

```json
{
    "code": 200,
    "message": "发布成功",
    "data": {
        "id": 1,
        "title": "Java开发工程师",
        // ... 其他职位信息
```

## 数据字典

### 职位状态
- ACTIVE：招聘中
- CLOSED：已结束
- DELETED：已删除

### 职位类型
- FULL_TIME：全职
- PART_TIME：兼职
- INTERN：实习

### 学历要求
- HIGH_SCHOOL：高中及以下
- JUNIOR_COLLEGE：大专
- BACHELOR：本科
- MASTER：硕士
- PHD：博士

### 工作经验
- NO_EXPERIENCE：无经验
- FRESH_GRADUATE：应届生
- 1_3_YEARS：1-3年
- 3_5_YEARS：3-5年
- 5_10_YEARS：5-10年
- ABOVE_10_YEARS：10年以上

### 企业规模
- BELOW_50：50人以下
- 50_200：50-200人
- 200_500：200-500人
- 500_1000：500-1000人
- 1000_5000：1000-5000人
- ABOVE_5000：5000人以上

### 学历
- 专科
- 本科
- 硕士
- 博士

### 性别
- 男
- 女
- 保密

## 认证相关接口

### 发送验证码
POST /api/auth/send-verification
请求体：
```json
{
    "email": "example@email.com"
}
```
返回：
```json
{
    "code": 200,
    "message": "验证码发送成功"
}
```

### 注册
POST /api/auth/register
请求体：
```json
{
    "email": "example@email.com",
    "code": "1234",
    "password": "yourpassword",
    "role": "STUDENT"  // STUDENT, COMPANY, ADMIN
}
```
返回：
```json
{
    "code": 200,
    "message": "注册成功",
    "data": {
        "username": "快乐小鹿001",
        "email": "example@email.com",
        "role": "STUDENT"
    }
}
```

### 登录
POST /api/auth/login
请求体：
```json
{
    "email": "example@email.com",
    "password": "yourpassword"
}
```
返回：
```json
{
    "code": 200,
    "message": "登录成功",
    "data": {
        "id": 1,
        "username": "快乐小鹿001",
        "email": "example@email.com",
        "role": "STUDENT",
        "token": "eyJhbGciOiJIUzI1NiJ9..."
    }
}
```

### 修改密码
POST /api/auth/change-password
请求体：
```json
{
    "email": "example@email.com",
    "code": "1234",
    "newPassword": "newpassword"
}
```
返回：
```json
{
    "code": 200,
    "message": "密码修改成功"
}
```

### 验证验证码
POST /api/auth/verify-code
请求体：
```json
{
    "email": "example@email.com",
    "code": "1234"
}
```
返回：
```json
{
    "code": 200,
    "message": "验证成功"
}
```

## 错误码说明

所有接口都会返回适当的HTTP状态码和统一格式的响应：

### HTTP状态码
- 200：成功
- 400：请求参数错误
- 401：未授权
- 403：禁止访问
- 404：资源未找到
- 500：服务器内部错误

### 响应格式
```json
{
    "code": 200,      // 业务状态码
    "message": "xxx", // 业务消息
    "data": {}        // 业务数据（可选）
}
```

### 业务状态码
- 200：成功
- 400：请求参数错误
  - 4001：验证码错误或已过期
  - 4002：邮箱已注册
  - 4003：用户名已存在
  - 4004：密码错误
- 401：未授权
  - 4011：token已过期
  - 4012：token无效
- 403：禁止访问
  - 4031：权限不足
- 404：资源未找到
  - 4041：用户不存在
- 500：服务器内部错误

## 认证说明

除了注册、登录、发送验证码和修改密码接口外，其他接口都需要在请求头中携带token：

```
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
```

token有效期为24小时，过期需要重新登录获取。

## 数据格式说明

### 用户角色
- STUDENT：学生
- COMPANY：企业
- ADMIN：管理员

### 用户状态
- ACTIVE：正常
- INACTIVE：未激活
- BLOCKED：已封禁

### 时间格式
所有时间字段都使用ISO 8601格式：`YYYY-MM-DDTHH:mm:ss.sssZ`

### 分页参数
支持分页的接口都接受以下参数：
- page：页码，从0开始
- size：每页条数

### 分页响应格式
```json
{
    "code": 200,
    "message": "success",
    "data": {
        "content": [],           // 数据列表
        "totalElements": 100,    // 总记录数
        "totalPages": 10,        // 总页数
        "size": 10,             // 每页条数
        "number": 0             // 当前页码
    }
}
```
